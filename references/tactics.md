# Additional high-leverage tactics

Most AI-driven launches miss these. Items marked **[ASK]** need user confirmation first.

1. **Pre-flight landing page + email capture.** Traffic to a repo with no capture leaks. If missing, stop and flag.

2. **UTM-tag every outbound link** before posting (see `intake-and-preflight.md` § 2b).

3. **Peak-hour scheduling, not fire-and-forget.**
   - HN: Tue–Thu, 6–9am US Pacific. One shot only.
   - Reddit: depends on sub, generally 8–11am local time of the sub's dominant audience.
   - LinkedIn: Tue–Thu, 7–9am or 12–1pm local.
   - TikTok / Reels: 6–10pm local.
   - Twitter: 9–11am or 7–9pm ET for tech Twitter.
   Use `/schedule` for out-of-window posts.

4. **Social-proof prime (first 60 min is everything).** Before HN / Product Hunt, line up 3–5 genuine engaged commenters from user's Discord / Slack / friends. **[ASK]** who to ping. Never buy/trade upvotes — rings get detected and ban the post.

5. **Flag high-comment-attraction posts.** After posting, tell the user which posts (HN, r/programming, top-tier subs) will likely attract comments in the first 2h so they can choose to check in. Agent doesn't monitor.

6. **Cross-link the graph.** Twitter thread → HN + YouTube. HN post text → Twitter. YouTube description → every social. Trust signal.

7. **Newsletter submissions.**
   - AI/LLM: TLDR AI, Ben's Bites, AlphaSignal, The Rundown
   - Dev tools: TLDR, Console.dev, Hacker Newsletter
   - Fintech/quant: **[ASK]** user which
   - Indie/SaaS: IndieHackers newsletter, SaaS Weekly, TheSaasNews

8. **Seeded influencer DMs (24–48h before public launch).** 3–5 mid-tier (10k–100k) niche accounts. DM preview with "no pressure, share if interested." **[ASK]** user to approve list + copy.

9. **Live-stream the launch.** 60–90min YouTube / Twitch / X live stream during post-out creates event feel. Optional, high-leverage.

10. **Second-wave content (days 2–7).**
    - Day 2: "Things I learned shipping X" (Twitter + LinkedIn)
    - Day 3: respond to top feedback → "what we're fixing"
    - Day 5: first user/result spotlight
    - Day 7: launch retro — numbers, screenshots, what worked

11. **Memes for meme-native audiences.** r/wallstreetbets / AI Twitter / dev-humor — a well-aimed meme beats 10 serious posts. Use `imagegen` or manual.

12. **Press / journalist outreach.** For serious projects (OpenHedge, TrustBench), pitch 5 journalists directly: who you are, what you built, why now, what's unique, ask. **[ASK]** user for outlet list.

13. **Waitlist / scarcity framing.** "First 100 users get X" / "closed beta — DM for access" converts 5–10× better than "try it free."

14. **Record the launch day itself.** Screen + webcam during post-out. Raw material for future "how I launched" content.

15. **OSS angle as a separate shot.** If repo is public, dedicated Show HN / r/opensource / r/programming pass JUST for the OSS angle, distinct from product launch.

16. **Post-mortem to the wiki.** After 48h, append to Obsidian wiki: what posted where, what hit, what flopped, what to repeat. Compounds.

17. **Don't use Buffer / Hootsuite / Typefully for Reddit / LinkedIn / TikTok.** They leave automation fingerprints and get flagged. Typefully/Hypefury are *acceptable* for Twitter threads only.

18. **Competitor piggyback.** Find a popular recent HN/Reddit thread about a competing product, drop a substantive comment linking user's project as "related". Only if genuinely value-adding.
